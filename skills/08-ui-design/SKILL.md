# Skill 08 - UI Design

## Tujuan

Mendesain antarmuka pengguna (UI) yang sederhana, mudah digunakan, dan sesuai dengan kebutuhan stakeholder.

---

## Kapan Digunakan

Digunakan setelah desain arsitektur, database, dan API selesai.

---

## Input

- docs/requirements/
- docs/design/01-architecture.md
- docs/design/02-database-api-design.md

---

## Langkah Kerja

1. Identifikasi halaman yang dibutuhkan.
2. Tentukan navigasi aplikasi.
3. Buat wireframe setiap halaman.
4. Tentukan komponen UI.
5. Dokumentasikan hasil.

---

## Output

- docs/design/03-ui-design.md
- docs/design/wireframe.md

---

## Aturan

- UI harus sederhana dan mudah dipahami.
- Semua fitur wajib harus memiliki halaman.
- Navigasi harus konsisten.

---

## Quality Check

- Semua halaman tersedia.
- Navigasi mudah digunakan.
- Form memiliki validasi.

---

## Kondisi Gagal

- Requirement belum lengkap.
- Database/API belum selesai.

---

## Human Review

Pastikan desain UI sesuai kebutuhan pengguna.

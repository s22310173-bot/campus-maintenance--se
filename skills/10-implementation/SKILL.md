# Skill 10 - Implementation

## Tujuan

Mengimplementasikan satu GitHub Issue menjadi kode yang berfungsi sesuai requirement.

---

## Kapan Digunakan

Digunakan setelah GitHub Issue dibuat.

---

## Input

- GitHub Issue
- Functional Requirement
- Database Design
- API Design
- UI Design

---

## Langkah Kerja

1. Baca GitHub Issue.
2. Identifikasi Requirement yang terkait.
3. Implementasikan hanya fitur pada Issue tersebut.
4. Jalankan aplikasi.
5. Periksa apakah Acceptance Criteria terpenuhi.
6. Commit perubahan ke branch yang sesuai.

---

## Output

- Source code
- Commit
- Pull Request

---

## Aturan

- Kerjakan satu Issue dalam satu branch.
- Jangan mengubah fitur di luar Issue.
- Ikuti coding style proyek.

---

## Quality Check

- Semua Acceptance Criteria terpenuhi.
- Tidak ada error saat dijalankan.
- Kode mudah dibaca.

---

## Kondisi Gagal

- Requirement belum jelas.
- Database atau API belum siap.

---

## Human Review

Periksa apakah implementasi sudah sesuai dengan GitHub Issue.

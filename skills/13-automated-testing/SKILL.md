# Skill 13 - Automated Testing

## Tujuan

Membuat dan menjalankan pengujian otomatis (Automated Testing) untuk memastikan seluruh fitur aplikasi bekerja sesuai requirement.

---

## Kapan Digunakan

Digunakan setelah implementasi fitur selesai.

---

## Input

- Source Code
- Functional Requirement
- Test Plan
- Acceptance Criteria

---

## Langkah Kerja

1. Identifikasi fungsi yang akan diuji.
2. Buat Unit Test.
3. Buat Integration Test.
4. Jalankan seluruh test.
5. Perbaiki test yang gagal.
6. Simpan hasil pengujian.

---

## Output

- File Unit Test
- File Integration Test
- Laporan hasil pengujian

---

## Aturan

- Setiap test harus memiliki tujuan yang jelas.
- Test harus dapat dijalankan berulang kali.
- Gunakan framework Vitest.

---

## Quality Check

- Semua test berhasil dijalankan.
- Tidak ada test yang gagal.
- Coverage meningkat.

---

## Kondisi Gagal

- Source code belum tersedia.
- Test tidak dapat dijalankan.

---

## Human Review

Periksa apakah hasil pengujian sesuai dengan requirement.

# Skill 11 - Code Review

## Tujuan

Memeriksa kualitas kode sebelum digabungkan ke branch utama (main).

---

## Kapan Digunakan

Dilakukan setelah implementasi satu GitHub Issue selesai dan sebelum Pull Request di-merge.

---

## Input

- Source Code
- GitHub Issue
- Functional Requirement
- Acceptance Criteria

---

## Langkah Kerja

1. Baca requirement yang terkait.
2. Bandingkan kode dengan requirement.
3. Periksa coding style.
4. Periksa kemungkinan bug.
5. Periksa apakah Acceptance Criteria sudah terpenuhi.
6. Berikan komentar jika ditemukan masalah.

---

## Output

docs/testing/01-code-review.md

---

## Aturan

- Jangan mengubah requirement.
- Fokus pada kualitas kode.
- Catat semua temuan.

---

## Quality Check

- Kode mudah dibaca.
- Tidak ada duplikasi kode.
- Tidak ada error yang terlihat.
- Acceptance Criteria terpenuhi.

---

## Kondisi Gagal

- Kode tidak dapat dijalankan.
- Requirement belum terpenuhi.

---

## Human Review

Reviewer memutuskan apakah Pull Request dapat di-merge.

# Skill 12 - Test Planning

## Tujuan

Menyusun rencana pengujian berdasarkan requirement sehingga seluruh fitur dapat diuji secara sistematis.

---

## Kapan Digunakan

Digunakan setelah tahap implementasi dan sebelum automated testing.

---

## Input

- Functional Requirement
- User Story
- Acceptance Criteria
- Source Code

---

## Langkah Kerja

1. Identifikasi fitur yang akan diuji.
2. Tentukan jenis pengujian.
3. Buat Test Case.
4. Hubungkan Test Case dengan Requirement.
5. Dokumentasikan hasil.

---

## Output

docs/testing/02-test-plan.md

---

## Aturan

- Semua Functional Requirement harus memiliki Test Case.
- Test Case harus dapat dijalankan.

---

## Quality Check

- Semua Requirement memiliki Test Case.
- Test Case mudah dipahami.

---

## Kondisi Gagal

- Requirement belum selesai.
- Acceptance Criteria belum lengkap.

---

## Human Review

Pastikan seluruh fitur telah memiliki rencana pengujian.

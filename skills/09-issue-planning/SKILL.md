# Skill 09 - Issue Planning

## Tujuan

Mengubah requirement menjadi GitHub Issues yang dapat dikerjakan satu per satu.

---

## Kapan Digunakan

Setelah tahap Design selesai.

---

## Input

- Functional Requirement
- UI Design
- Database Design

---

## Langkah Kerja

1. Baca Functional Requirement.
2. Kelompokkan pekerjaan.
3. Buat GitHub Issue.
4. Tentukan prioritas.
5. Tentukan estimasi.

---

## Output

docs/planning/issue-plan.md

---

## Aturan

- Satu Issue hanya satu pekerjaan.
- Issue harus mengacu pada Requirement ID.
- Issue harus memiliki Acceptance Criteria.

---

## Quality Check

- Semua Requirement memiliki Issue.
- Tidak ada Issue yang tumpang tindih.

---

## Human Review

Periksa apakah Issue sudah siap dikerjakan.
